# WM-811K Wafer Map Failure Analysis

End-to-end pipeline: from the raw WM-811K pickle dataset to an LLM-based baseline evaluation.

---

## Project Structure

```
wm811k/
├── data/
│   ├── source/
│   │   └── LSWMD.pkl                    # raw dataset (download manually)
│   ├── datasets/
│   │   └── wm811k_sample_1800.parquet   # generated by build_dataset.py
│   └── wafer_images/                    # generated by build_dataset.py
│       ├── center/
│       ├── donut/
│       ├── edge-loc/
│       ├── edge-ring/
│       ├── loc/
│       ├── near-full/
│       ├── none/
│       ├── random/
│       └── scratch/
├── scripts/
│   ├── pickle_reader.py     # standalone: pickle → parquet
│   ├── save_as_imgs.py      # standalone: parquet → images
│   ├── parquet_reader.py    # standalone: show parquet info
│   └── prompts/
│       └── baseline.yaml    # few-shot prompt template
```

## Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Download the Raw Dataset

Download `LSWMD.pkl` from [Kaggle — WM-811K Wafer Map](https://www.kaggle.com/datasets/qingyi/wm811k-wafer-map) and place it at:

```
data/source/LSWMD.pkl
```

### 3. Configure API Keys

Create a `.env` file (or edit the existing one):

```
OPENAI_API_KEY=sk-...
EVA_MODEL=gpt-4o-mini
```

## Pipeline

### Step A — Build Dataset (pickle → parquet + images)

```bash
python build_dataset.py
```

This single script runs three stages in sequence:

| Stage | Input | Output | Description |
|-------|-------|--------|-------------|
| 1 | `data/LSWMD.pkl` | `data/datasets/wm811k_sample_1800.parquet` | Cleans labels, samples up to 200 per class (9 classes → ≤1800 rows) |
| 2 | parquet | `data/wafer_images/{class}/*.png` | Renders each wafer map as a grayscale PNG |
| 3 | parquet | stdout | Prints head, shape, columns, and class distribution |

Sample output:

```
[1/3] Reading pickle: .../data/LSWMD.pkl
    Saved: .../data/datasets/wm811k_sample_1800.parquet  shape=(1800, 6)
    center       200
    donut        200
    ...

[2/3] Saving wafer map images to: .../data/wafer_images
    Done — 1800 images written.

[3/3] Parquet preview: .../data/datasets/wm811k_sample_1800.parquet

  Head (5 rows, non-waferMap columns):
  lotName  waferIndex failureType trianTestLabel  ...

  Shape : (1800, 6)
  Columns: ['lotName', 'waferIndex', 'failureType', 'trianTestLabel', 'dieSize', 'waferMap']

  Class distribution:
  center       200
  donut        200
  edge-loc     200
  ...
```

---

### Step B — Baseline Evaluation

```bash
python baseline_evaluation.py
```

Sends the first 5 wafer map images from the parquet to an OpenAI vision model using the prompt template in `scripts/prompts/baseline.yaml`.

The prompt asks the model to classify each wafer into one of the 9 defect classes and return structured JSON:

```json
{
  "type": "center",
  "reasoning": "Defects are concentrated in the central region of the wafer, consistent with a center-type failure pattern."
}
```

Output example:

```
[center] lot123_wafer5
{"type": "center", "reasoning": "..."}

[edge-ring] lot456_wafer2
{"type": "edge-ring", "reasoning": "..."}
```

---

## Defect Classes

| Class | Description |
|-------|-------------|
| `center` | Defects concentrated in the wafer center |
| `donut` | Center normal; ring-shaped defective band |
| `edge-loc` | Localized defects near the wafer edge |
| `edge-ring` | Continuous ring defects near the edge |
| `loc` | Localized clustered defects anywhere |
| `near-full` | Almost full-wafer defect coverage |
| `none` | Clean wafer / negligible defects |
| `random` | Scattered, randomly distributed defects |
| `scratch` | Linear / streak-like defect pattern |

## Running Individual Scripts

Each script in `scripts/` can also be run standalone from the project root:

```bash
# pickle → parquet only
python scripts/pickle_reader.py

# parquet → images only
python scripts/save_as_imgs.py

# show parquet info only
python scripts/parquet_reader.py
```

## Notes

- All paths are resolved relative to each script's own location using `pathlib`, so scripts work correctly regardless of which directory you run them from.
- The `.env` file is not committed. Do not share your API key.
- `data/` is excluded from git (see `.gitignore`). Download `data/source/LSWMD.pkl` from Kaggle before running the pipeline.
